01 — The Problem

The governance gap
behind the software request

Bangladesh's OMS program serves millions of vulnerable citizens daily. Without a real-time verification layer, four structural vulnerabilities persist — and the government cannot currently distinguish legitimate distribution from systematic leakage.

01 / 04
🔄
Duplicate Lifting
A beneficiary can collect food grain multiple times in a single day at different centers — with no technical barrier to prevent it.
02 / 04
📉
Monthly Quota Leakage
Without real-time quota tracking across the network, beneficiaries exceed monthly entitlements across multiple visits — undetected.
03 / 04
📴
Offline Accountability Gap
Rural dealers operate without system access during connectivity outages — creating completely unrecorded distribution windows.
04 / 04
📊
Zero Policy Intelligence
Allocation decisions are based on estimates, not real consumption data — causing chronic mismatches between supply and actual demand.

04 — Offline-First Architecture

Designed for
Bangladesh's reality

The SRS requires local data with conditional sync on 2G networks. This is not a feature toggle — it is a fraud prevention mechanism. A system that records locally but validates centrally cannot be gamed by deliberately working offline.

Dealer selects beneficiary from local cached list — no server call needed
App checks offline eligibility snapshot — quota and daily lifting status
Face captured with real-time guidance — lighting, position, distance
Biometric match runs against locally cached model — no internet required
Transaction recorded instantly in local SQLite — guaranteed no data loss
Exception flow if uncertain — photo + reason code + officer log
Auto-sync to central server when connectivity is detected
Server validates, deduplicates, audits — retroactive enforcement catches all fraud

Minimum connectivity supported

SYNC ARCHITECTURE
· UUID transaction IDs — globally unique, prevents duplicate processing
· Idempotent APIs — safe to retry any sync without side effects
· Conflict resolution — central server wins on quota/duplicate checks
· Anomaly detection — unusual delayed bulk syncs trigger alerts
